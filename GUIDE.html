Step 1: Get Your Server

You need a cloud server (VPS) to run OpenClaw. We'll use Hostinger for this workshop, but any Ubuntu 24.04 VPS works.

üìù Sign Up for Hostinger VPS

Create an account at bit.ly/ClawsReady2

Choose a VPS plan ‚Äî the KVM 2 plan (2 vCPU, 8GB RAM) is sufficient to start. Billing is flexible: 1 month or 12 months, with discounts based on your commitment level.

Select a region close to you for lower latency

Choose Ubuntu 24.04 as the operating system

Add your SSH key during setup (recommended) or use password auth initially

üí° SSH Key Setup

If you don't have an SSH key, generate one:

mkdir -p ~/.ssh && cd ~/.ssh
ssh-keygen -t ed25519 -C "your@email.com"

Then copy your public key:

cat id_ed25519.pub

Paste this into Hostinger's SSH key section during VPS setup.

üîå Connect to Your Server

Once your VPS is running, connect via SSH:

ssh root@YOUR_VPS_IP_ADDRESS

If you set up SSH keys, you'll connect automatically. Otherwise, enter your root password.

üéØ Goal: You should now be SSH'd into your Hostinger VPS, seeing a shell prompt like root@hostname:~#

Step 2: Install OpenClaw

The install script handles everything: Node.js, dependencies, and the onboarding wizard.

üöÄ Run the Install Script

Paste this into your VPS terminal:

curl -fsSL https://openclaw.ai/install.sh | bash

This will take a few minutes. Once complete, you'll have the openclaw command available.

‚öôÔ∏è If Node.js Wasn't Installed

If the install script didn't provision Node.js, run this fallback:

curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v && npm -v

üéØ Run Onboarding

Start the configuration wizard:

openclaw onboard --install-daemon

This will:

  • Configure your AI model provider (we'll do this in Step 3)
  • Set up authentication
  • Configure gateway settings
  • Install the daemon service (runs in background)
üéØ Goal: You should reach the onboarding wizard prompts. Don't complete them yet ‚Äî we'll configure the model in Step 3.

Step 4: Setup Discord

Connect OpenClaw to Discord so you can chat with your agent from anywhere.

ü§ñ Create a Discord Bot

Go to Discord Developer Portal

Click New Application and give it a name (e.g., "My Claw")

Navigate to Bot in the left sidebar

Click Add Bot, then Reset Token

Copy the token ‚Äî you'll paste this during onboarding

üîó Invite Bot to Your Server

In Developer Portal, go to OAuth2 ‚Üí URL Generator

Check the bot scope

Under Bot Permissions, select:

  • Read Messages/View Channels
  • Send Messages
  • Read Message History

Copy the generated URL and open it in your browser

Select your server and authorize the bot

üîê Enable Message Content Intent

Required for your bot to read your messages:

In Developer Portal, go to Bot

Scroll to Privileged Gateway Intents

Enable Message Content Intent

Save changes

üì± DM Your Bot

To send direct messages to your bot:

In Developer Portal, go to General Information

Copy the Application ID

Navigate to https://discord.com/users/<APPLICATION_ID>

Click Message to start a DM

‚úÖ Complete Onboarding

Back in your VPS terminal, continue the onboarding:

  • Select Discord as a channel
  • Paste your bot token
  • When prompted, DM your bot ‚Äî it will reply with a pairing code
  • Approve the pairing:
openclaw pairing approve discord <PAIRING_CODE>
üéØ Goal: Send a DM to your bot on Discord and receive a response. Your agent is live!

Troubleshooting

‚ùå openclaw: command not found
Restart your shell session or run source ~/.bashrc (or ~/.zshrc). If that doesn't work, verify the install path is in your PATH.
‚ùå Gateway not running
Rerun onboarding with daemon install: openclaw onboard --install-daemon
‚ùå Gateway unreachable
Verify host/port in config, check local firewall rules: sudo ufw status
‚ùå Bot doesn't respond to DMs
  1. Check Message Content Intent is enabled
  2. Verify the bot token is correct in config
  3. Check pairing was approved: openclaw pairing list
‚ùå Channel test fails
Confirm channel config and destination target in openclaw config
